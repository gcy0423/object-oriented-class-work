# v11-teacher-ai-first-frontend

v11 目标是基于 `prototypes/teacher-ai-first` 重写新版教师前端，让教师端从旧工作台的功能聚合形态，升级为稳定、清晰、有呼吸感的 AI-first 教师工作台。v10 已经完成教师读取学生 AI 证据和干预的基础联动；v11 重点处理教师端真实使用体验、页面层级、布局稳定性和上下文 AI 助手。

## 原型依据

- 原型目录：`prototypes/teacher-ai-first`
- 实现必须以该原型的教师端信息架构、导航关系、页面主次层级和 AI 助手位置为基准。
- 可以根据真实后端能力补齐功能，但不能把旧版教师工作台直接搬进新页面。
- 如果原型和后端功能冲突，以“原型体验优先、后端能力补齐”的方式做适配，并在实现记录中说明差异。

## 版本目标

- 基于 `prototypes/teacher-ai-first` 重写教师端主要前端页面。
- 教师端具备稳定 shell、侧栏、顶部上下文、主内容区和右侧 AI 助手。
- 教师可在新版页面中完成课程、学生、作业、练习、批改、学情、干预等核心工作流。
- 教师端右侧 AI 助手随当前页面上下文变化，而不是固定公告栏。
- UI 必须体现呼吸感、稳定感和清楚的信息层级。
- 页面不能暴露内部 ID，例如 `course_ood`、`user_student`。
- 首屏和路由切换不能闪回旧教师工作台。
- 窄屏下布局仍然左对齐、有间距、无横向溢出。

## 设计硬约束

本版本实现时必须把以下清单当作硬验收：

- 模块之间必须有明确间距。
- 卡片内部必须有 padding。
- 同类操作必须左对齐自然排布。
- 用户界面不能显示内部 ID。
- 侧栏高度、滚动和底部用户卡必须稳定。
- 首屏不能闪旧界面。
- 页面标题、卡片标题、正文、按钮必须有层级。
- 按钮和文字不能互相挤压。
- 窄屏换行后仍要左对齐且保持 gap。
- AI 上下文助手必须随页面变化。

## 本版本不做

- 不重写学生端 AI-first 页面。
- 不新增大规模后端业务域。
- 不引入 React/Vue/构建链，继续使用原生 ESM。
- 不改变 v7-v10 已经完成的学生端 AI 工作流接口。
- 不把内部 ID 改为业务主键；只要求展示层隐藏内部 ID。
