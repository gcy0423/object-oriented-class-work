# Acceptance

## 功能验收

- [ ] 新版教师端页面基于 `prototypes/teacher-ai-first` 实现。
- [ ] 原型中的核心导航、页面层级和 AI 助手布局被保留。
- [ ] 教师登录后进入新版教师首页。
- [ ] 教师端所有新版页面使用统一 teacher shell。
- [ ] 左侧栏在不同页面高度和布局稳定。
- [ ] 退出登录清楚可见。
- [ ] 教师可查看课程学情。
- [ ] 教师可查看学生画像。
- [ ] 教师可查看学生 AI 结果和 AI 时间线。
- [ ] 教师可查看提交 AI 自检证据。
- [ ] 教师可发送干预提醒。
- [ ] 右侧 AI 助手随页面上下文变化。

## UI 验收

- [ ] section 和 section 之间不贴在一起。
- [ ] 卡片和卡片之间有稳定 gap。
- [ ] 标题、正文、按钮之间有稳定间距。
- [ ] 卡片内部有 padding，文字不贴边。
- [ ] 列表卡片的标题、描述、状态、按钮不堆成一坨。
- [ ] tab 和按钮组左对齐自然排布。
- [ ] 不滥用 `space-between` 把按钮甩到两端。
- [ ] 页面标题、卡片标题、正文、按钮层级清楚。
- [ ] 小卡片不使用过大的标题。
- [ ] 按钮之间有 gap。
- [ ] 按钮和上方文字有距离。
- [ ] 换行后仍有间距。

## 稳定性验收

- [ ] 首屏加载不闪旧教师工作台。
- [ ] 路由切换不疯狂闪烁。
- [ ] 数据刷新时 shell 不重排。
- [ ] 侧栏底部用户卡不跳动。
- [ ] 窄屏下无横向溢出。
- [ ] 窄屏下按钮换行后仍左对齐。

## 数据展示验收

- [ ] UI 不展示 `course_ood`。
- [ ] UI 不展示 `user_student`。
- [ ] UI 不展示 `assignment_*`、`submission_*` 等内部 ID。
- [ ] 缺少名称时显示可读 fallback。
- [ ] 内部 ID 只保留在 `data-id` 或请求参数中。
