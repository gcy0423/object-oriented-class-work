# Scope

## 做什么

- 以 `prototypes/teacher-ai-first` 为新版教师端前端的设计和交互基准。
- 新增教师端 AI-first 路由和 shell。
- 梳理教师端页面结构：课程、学生、作业、练习、批改、学情、干预、报告。
- 将教师端页面从旧版通用 layout 迁移到新版教师专属 layout。
- 建立教师端右侧上下文 AI 助手模型。
- 在教师端使用可读名称展示课程、学生、作业和班级。
- 修复教师端页面中的拥挤、贴边、无间距、过度 `space-between` 问题。
- 确保侧栏在不同教师页面之间保持稳定。
- 确保教师端首屏加载时不闪旧界面。
- 建立窄屏适配规则。

## 不做什么

- 不脱离 `prototypes/teacher-ai-first` 自行另起一套教师端信息架构。
- 不重写服务端认证、课程、作业、评分、AI 工作流的核心逻辑。
- 不做复杂权限矩阵 UI。
- 不做移动端独立 App。
- 不替换现有数据文件结构，除非展示名映射确实缺字段。
- 不在 UI 中展示调试字段、数据库字段或内部 ID。

## 页面范围

页面范围来自 `prototypes/teacher-ai-first`，实现时允许按后端真实能力命名和拆分，但不能遗漏原型中的核心教师工作流。

- 教师首页/总览。
- 课程学情页。
- 学生画像页。
- 作业与批改页。
- 练习与错题页。
- 干预队列页。
- 报告/导出页。
- 设置/用户信息的教师端入口。
