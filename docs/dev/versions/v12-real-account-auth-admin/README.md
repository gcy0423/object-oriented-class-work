# v12-real-account-auth-admin

v12 承接 v11 的新版教师端前端。当前系统仍保留“邮箱 + 角色自动建号”的轻量登录方式，管理员端也仍有旧 `AI` 页面入口。v12 的目标是把身份系统升级为真实账号与鉴权系统，并把管理员端从旧 AI 页面切换到账号、角色和权限管理。

本版本完成后，用户必须通过已注册账号和密码登录；学生可以自助注册，教师和管理员由管理员创建或提权；管理员端可以管理账号、状态、密码重置和权限矩阵。

## 版本目标

- 登录改为 `email + password`，不再允许登录时自动创建用户。
- 登录页新增注册入口，开放注册默认创建学生账号。
- 密码只以哈希和盐保存，任何接口不得返回明文密码。
- Bearer Token 继续作为前端和 Gateway 的会话凭证。
- `sessions` collection 记录会话状态，支持退出登录和 token 失效。
- 管理员可创建用户、编辑角色和状态、重置密码。
- 管理员可查看账号最近登录和会话状态。
- 管理员可维护角色权限矩阵。
- 管理员端删除旧 `AI` 页面入口，账号系统管理成为管理端主入口。
- 保持学生端和教师端既有业务接口兼容。

## 本版本不做

- 不引入外部账号体系、OAuth、短信验证码或邮件验证码。
- 不做多租户、多学校组织隔离。
- 不引入外网依赖或重型认证框架。
- 不把所有业务服务改成直接读账号库，仍通过 Gateway 注入用户上下文。
- 不实现复杂细粒度 ABAC 权限策略。
- 不改写学生端和教师端 AI-first 主流程。
- 不删除后端 AI 能力，只删除管理员端旧 `ai` 页面入口。

## 新增能力摘要

| 能力 | 目标 |
| --- | --- |
| 密码登录 | 用户必须用邮箱和密码登录 |
| 开放注册 | 注册默认创建学生账号 |
| 密码哈希 | 使用标准库哈希和盐保存密码 |
| 会话管理 | token 可被退出登录失效 |
| 账号管理 | 管理员创建、编辑、禁用、归档账号 |
| 密码重置 | 管理员可重置用户密码 |
| 权限矩阵 | 管理员可查看和编辑角色权限 |
| 管理端入口 | 删除旧 AI 页面，新增账号与权限主入口 |

## 目标启动命令

```bash
npm run start:services:mock
npm run test:services
npm test
```

## 版本完成标准

- 未注册邮箱不能通过登录自动建号。
- 注册成功后可使用密码登录。
- 错误密码、禁用账号、归档账号不能登录。
- 退出登录后旧 token 不能访问 `/api/me`。
- 管理员可创建教师和管理员账号。
- 管理员可修改用户角色、状态和基础资料。
- 管理员可重置密码，旧密码立即失效。
- 管理员可查看并保存角色权限矩阵。
- 管理端导航不再展示旧 `AI` 页面入口。
- `npm test` 和 `npm run test:services` 通过。
