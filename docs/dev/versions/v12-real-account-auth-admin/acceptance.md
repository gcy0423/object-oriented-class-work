# Acceptance

## 鉴权验收

- [ ] 登录必须输入邮箱和密码。
- [ ] 登录不再根据邮箱自动创建用户。
- [ ] 开放注册默认创建学生账号。
- [ ] 注册接口不能创建教师或管理员。
- [ ] 密码不以明文保存。
- [ ] 用户响应不包含 `passwordHash` 或 `passwordSalt`。
- [ ] 错误密码无法登录。
- [ ] 非 active 用户无法登录。
- [ ] 登录成功后 token 可访问 `/api/me`。
- [ ] 退出登录后旧 token 失效。
- [ ] token 过期后无法访问受保护接口。

## 管理员账号验收

- [ ] 管理员可以查看账号列表。
- [ ] 管理员可以按角色、状态、关键词筛选账号。
- [ ] 管理员可以创建学生、教师和管理员账号。
- [ ] 管理员可以编辑用户角色和状态。
- [ ] 管理员可以重置用户密码。
- [ ] 重置密码后旧密码不能登录。
- [ ] 禁用或归档用户后该用户现有 session 失效。
- [ ] 非管理员不能创建高权限账号。
- [ ] 非管理员不能重置他人密码。

## 权限矩阵验收

- [ ] 管理员可以查看角色权限矩阵。
- [ ] 管理员可以编辑角色权限矩阵。
- [ ] 权限矩阵保存后刷新不丢。
- [ ] 系统保留 student、teacher、admin 三类基础角色。
- [ ] admin 保留账号和权限管理能力。

## UI 验收

- [ ] 登录页包含密码字段。
- [ ] 登录页提供注册入口。
- [ ] 注册页或注册模式不要求选择角色。
- [ ] 管理员导航不展示旧 `AI` 页面入口。
- [ ] 管理入口名称体现账号和权限管理。
- [ ] 账号列表展示状态、角色、邮箱、最近登录。
- [ ] 创建账号、重置密码、权限矩阵操作清楚可见。
- [ ] 窄屏下表单和表格无横向溢出。

## 回归验收

- [ ] 学生端 AI-first 页面仍可通过学生账号访问。
- [ ] 教师端 AI-first 页面仍可通过教师账号访问。
- [ ] Gateway 仍能向各服务注入用户上下文。
- [ ] `npm test` 通过。
- [ ] `npm run test:services` 通过。
