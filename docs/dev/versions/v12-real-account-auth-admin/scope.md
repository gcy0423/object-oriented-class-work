# Scope

## 做什么

- 将 `POST /api/auth/login` 从自动建号改为真实密码登录。
- 新增开放注册流程，默认注册学生账号。
- 为 identity-service 增加密码哈希、密码校验、会话记录、退出登录和 token 撤销能力。
- 为 Gateway 补齐注册、退出登录和管理员账号管理代理接口。
- 扩展 `identity-admin` 页面为账号与权限管理中心。
- 管理员端支持创建账号、编辑角色/状态、重置密码、查看最近登录。
- 管理员端支持读取和编辑角色权限矩阵。
- 从旧 shell 导航中删除 `ai` 页面入口。
- 登录页显示密码输入、注册入口和明确错误提示。
- 保持现有学生端、教师端和服务间用户上下文传递方式。

## 不做什么

- 不新增短信、邮箱验证码、第三方登录或 OAuth。
- 不新增真实邮件发送或密码找回邮件。
- 不做生产级密码复杂度策略，只做基础长度和必填校验。
- 不实现 IP 风控、设备指纹或登录地展示。
- 不把管理员权限拆成复杂的逐按钮授权。
- 不删除 ai-service、学生 AI 页面或教师 AI-first 助手。
- 不迁移到数据库服务，继续使用当前 JSON 数据文件。

## 页面范围

- 登录页：登录表单、注册表单、错误提示、注册成功后引导登录。
- 管理员端账号与权限页：账号列表、筛选、创建账号、用户详情、重置密码、权限矩阵。
- 旧 shell 导航：移除 `ai` route 的可见入口。
- 其他学生端和教师端页面只做登录态兼容，不做重写。
