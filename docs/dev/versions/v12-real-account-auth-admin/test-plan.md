# Test Plan

## 自动测试

```bash
npm test
npm run test:services
```

## 服务测试

- 注册学生账号成功。
- 重复邮箱注册失败。
- 登录必须提供邮箱和密码。
- 错误密码返回 401。
- 未注册邮箱不能自动建号。
- `inactive`、`archived`、`pending` 用户不能登录。
- 登录成功返回 `user` 和 `token`。
- `/api/me` 可用有效 token 读取当前用户。
- logout 后旧 token 不能访问 `/api/me`。
- 重置密码后旧密码不能登录，新密码可登录。
- 用户禁用或归档后旧 session 失效。
- 非管理员不能创建教师或管理员账号。
- 管理员可以创建教师和管理员账号。
- 管理员可以更新用户状态、角色和资料。
- 管理员可以读取和更新权限矩阵。

## 前端模块测试

- `ApiClient.login` 发送 email/password。
- `ApiClient.register` 调用 `/api/auth/register`。
- `ApiClient.logout` 调用 `/api/auth/logout`。
- 管理员账号相关方法路径正确。
- 登录页渲染密码字段。
- 注册模式默认学生角色。
- 旧 `ai` route 不在管理员导航中。
- `identityAdminView` 渲染账号创建、重置密码、权限矩阵区域。

## 浏览器验证

桌面视口：

- 1440 x 900
- 1280 x 800

窄屏视口：

- 390 x 844
- 768 x 1024

验证项：

- 登录和注册表单不溢出。
- 错误提示清楚显示。
- 管理员导航不出现 `AI`。
- 账号与权限页首屏能看到账号列表和关键操作。
- 创建账号、重置密码、权限矩阵表单按钮不互相挤压。
- 窄屏下管理表单自然换行。

## 手工工作流

- 注册一个学生账号。
- 使用学生账号登录。
- 学生退出登录。
- 使用管理员账号登录。
- 创建教师账号。
- 创建管理员账号。
- 修改学生账号为 inactive 并确认无法登录。
- 重置教师密码并用新密码登录。
- 编辑权限矩阵并刷新后确认保存。
- 确认旧 `AI` 页面入口不可见。
