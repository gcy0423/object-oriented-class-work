# v9-student-ai-first-phase3-complete-student-loop

v9 第三阶段承接 v8 的正式学生端 AI 工作流。v7 让 `student-ai-first-v2` 原型真实可用，v8 让 AI 能力后端化；v9 的目标是把学生端剩余主体验一次性收口：AI 结果持久化、行动卡片状态、任务草稿、提交草稿、真实附件上传、笔记 CRUD、通知提醒、AI 质量评测和知识图谱联动。

本阶段完成后，除教师端联动外，学生端应基本覆盖 `student-ai-first-v2` 原型和真实使用所需的关键闭环。

## 版本目标

- 保存和恢复 AI 今日建议、薄弱点分析、作业拆解、提交自检、笔记整理结果。
- AI 行动卡片支持完成、忽略、转任务、转练习、打开目标页面。
- AI 任务草稿支持后端保存、编辑、确认创建任务、删除。
- 作业提交草稿支持后端保存、刷新恢复、AI 自检关联、一键正式提交。
- 实现真实附件上传服务，提交作业可关联上传文件。
- 笔记支持列表、详情、编辑、删除、搜索和 AI 整理结果保存。
- 学生端接入通知和提醒：截止提醒、AI 建议提醒、草稿未提交提醒。
- 增加学生 AI 学习时间线，展示最近 AI 建议、任务、提交、自检、练习和笔记事件。
- 增加 AI 质量评测样例集和 prompt 回归测试。
- 强化知识图谱/知识库联动：错题、练习、笔记、作业建议可引用知识点。
- 保持 v7/v8 学生端页面和接口兼容。

## 本版本不做

- 不实现教师端联动；该范围放到 v10。
- 不做生产级对象存储；本阶段文件上传可使用本地 workspace/data/uploads 或等价本地目录。
- 不实现复杂富文本编辑器。
- 不实现多人实时协同编辑。
- 不新增外网依赖。
- 不要求真实模型在线评测；AI 质量评测使用 mock/provider fixture 和结构化输出样例。

## 新增能力摘要

| 能力 | 目标 |
| --- | --- |
| AI 结果持久化 | 刷新后恢复最近 AI 建议和整理结果 |
| 行动卡片状态 | 完成、忽略、转任务、转练习 |
| 任务草稿 | 保存、编辑、确认、删除 |
| 提交草稿 | 保存、恢复、自检、正式提交 |
| 文件上传 | 作业提交附件真实上传 |
| 笔记 CRUD | 列表、详情、编辑、删除、搜索 |
| 通知提醒 | 截止、草稿、AI 建议提醒 |
| 学习时间线 | 汇总 AI 和学习过程事件 |
| AI 质量评测 | prompt/schema/fallback 回归 |
| 知识图谱联动 | 知识点证据进入 AI 建议 |

## 目标启动命令

```bash
npm run start:services:mock
npm run test:services
npm test
```

## 版本完成标准

- AI 学习台刷新后恢复最近 daily plan。
- AI 行动卡片状态可保存并恢复。
- AI 任务草稿可保存、编辑、确认创建、删除。
- 作业提交草稿刷新不丢。
- 作业附件可上传并随 submission 提交。
- 笔记可列表查看、编辑、删除、搜索。
- AI 笔记整理结果可历史查看并保存为新笔记。
- 学生端可看到通知、提醒和 AI 学习时间线。
- AI 质量评测测试通过。
- 知识点证据能进入薄弱点分析、练习建议和作业指导。
- `npm test` 和 `npm run test:services` 通过。
